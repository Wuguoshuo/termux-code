# Skill 6: 使用个人素材库

> **降 AI 味的核心**：从个人素材库搜索真实经历、观点、案例
>
> 方法 A：直接搜索原始数据（使用 Grep）
> 方法 B：查看已提炼素材（主题索引）

---

## 触发时机

1. Skill 7 创作章节初稿前（准备阶段）
2. 章节写作过程中（需要情感/观点/场景时）
3. Skill 8 审校时（增加真实感）

---

## 核心目的

**为什么使用个人素材库？**

- ✅ 降 AI 味：用真实经历替代抽象描写
- ✅ 增加人味：具体场景、真实感受
- ✅ 提升可信度：真实案例支撑观点
- ✅ 保持个性：用作者真实的表达方式

**素材库位置**: `materials/`

**素材库结构**:
- `原始素材/` - 原始记录
- `经历索引/` - 真实经历分类
- `观点索引/` - 个人观点和想法
- `金句库/` - 喜欢的句子、修辞手法

---

## 工作流程

### Step 1: 判断使用场景

**Think Aloud 示例**:
```
要写第3章，主角失恋...

我思考：
- 这是情感描写场景
- 需要真实的情感感受
- 应该搜索素材库

我的计划：
1. 生成搜索关键词："失恋 | 分手 | 痛苦 | 心碎"
2. 在 `materials/经历索引/` 搜索
3. 在 `materials/金句库/` 搜索
4. 整理找到的素材
5. 在写作时自然融入
```

**常见使用场景**:

#### 场景 1: 情感描写
- 需要搜索：失恋、分手、痛苦、心碎、思念、遗憾等
- 素材来源：`经历索引/`、`金句库/`

#### 场景 2: 观点表达
- 需要搜索：失败、成功、成长、友情、爱情等
- 素材来源：`观点索引/`、`金句库/`

#### 场景 3: 对话设计
- 需要搜索：真实对话、说话风格
- 素材来源：`原始素材/`、`经历索引/`

#### 场景 4: 结尾升华
- 需要搜索：关于失败、关于成长、关于人生的观点
- 素材来源：`观点索引/`、`金句库/`

### Step 2: 方法 A - 直接搜索原始数据（推荐）

**适用情况**:
- 有明确的原始素材文件
- 需要查找特定场景、对话
- 素材较多，主题索引覆盖不全面

**工具**: Grep（全文搜索）

**搜索示例**:

```bash
# 搜索失恋相关素材
grep -r "失恋\|分手\|痛苦\|心碎" materials/经历索引/

# 搜索关于失败的观点
grep -r "失败" materials/观点索引/

# 搜索心碎比喻
grep -r "心碎" materials/金句库/
```

**搜索输出**:
```markdown
## 素材搜索结果

### 关键词：失恋 | 分手 | 痛苦 | 心碎

#### 来自经历索引/

1. **文件**: `materials/经历索引/失恋-分手.md`
   - 经历 1：2022年分手
     - 真实感受：震惊、不信、愤怒、接受
     - 对话示例："我们...不适合..."
     - 可用于：失恋场景、心碎感受

2. **文件**: `materials/经历索引/心碎回忆.md`
   - 经历 1：深夜想起前任
     - 真实感受：怀念、遗憾
     - 场景描述：[具体场景]
     - 可用于：失恋后的思念场景

#### 来自金句库/

1. **文件**: `materials/金句库/比喻-心碎.md`
   - 表达 1："心里堵得慌，像有什么东西碎了"
   - 表达 2："精心准备的礼物被扔进垃圾桶"
   - 可用于：失望情绪、心碎描写
```

### Step 4: 整理和使用素材

**使用原则**:

#### 1. 真实是核心
- ✅ 所有素材都是真实的
- ❌ 不能编造或夸大
- ❌ 不能借用他人经历

#### 2. 可以改写成长文逻辑
- ✅ 原始素材可能是碎片化记录
- ✅ 写作时改写成完整段落
- ✅ 保持真实核心

#### 3. 自然融入
- ❌ 不要生硬插入
- ✅ 要和情节、人物、场景自然结合
- ✅ 不是简单粘贴

**使用示例**:

```markdown
## 使用示例 1: 情感描写

### 原始素材
```
素材库 - 经历 1：
真实感受：震惊、不信、愤怒、接受
```

### 融入章节

❌ 不好的融入：
林凡感到震惊和不信，以及愤怒，最后接受。

✅ 好的融入：
她说了那句话后，林凡脑子空了，像是被人抽了一棍子。

接着是愤怒。凭什么？凭什么说断就断？

等他冷静下来，心里堵得慌，像有什么东西碎了。

---

## 使用示例 2: 对话设计

### 原始素材
```
素材库 - 经历 1：
对话示例："我们...不适合..."
```

### 融入章节

❌ 不好的融入：
她说，我们不适合。

✅ 好的融入：
"我们..."她低下头，声音越来越小，"...不适合。"

---

## 使用示例 3: 结尾升华

### 原始素材
```
素材库 - 观点索引：
核心观点："失败不可怕，可怕的是不敢承认"
```

### 融入章节

❌ 不好的融入：
这次失败让他意识到，失败不可怕。

✅ 好的融入：
林凡坐在那里，手里握着时间机器的图纸。

失败吗？可能是。

但那又怎样呢？

他想起了父亲常说的话："失败不可怕，可怕的是不敢承认。"

他笑了。
```

## 特殊情况处理

### 情况 1: 搜索结果为空

**处理**:
- 说明素材库中无相关内容
- 建议用户：
  1. 添加相关素材
  2. 或使用 AI 通用描写（但标记为"待替换"）
- 继续写作

**示例**:
```
搜索"时间旅行"相关素材...

结果：素材库中无相关内容

处理：
- 说明："素材库中没有时间旅行相关的经历"
- 建议：
  1. 您可以在 `materials/经历索引/` 添加时间旅行的思考
  2. 或我先使用通用描写，您后续可以替换
- 继续写作
```

### 情况 2: 素材不完全匹配

**处理**:
- 选择最接近的素材
- 说明素材和场景的差异
- 改写调整以适应场景
- 或组合多个素材

**示例**:
```
章节需要"失恋的愤怒"

找到素材：
- 素材 A：失恋的痛苦（感受为主）
- 素材 B：愤怒的情绪（但不是失恋场景）

处理：
- 使用素材 A 的感受
- 调整为"愤怒"的语境
- 改写成完整场景
```

### 情况 3: 用户要求不使用素材

**处理**:
- 尊重用户选择
- 不搜索和使用素材库
- 但在日志中记录"跳过素材使用"
- 继续写作

## 常见错误避免

❌ **不要这样做**:
- 不搜索直接编造
- 素材不相关硬使用
- 直接复制粘贴原始素材
- 素材和场景完全不匹配强使用
- 编造或夸大素材
- 不记录使用日志

✅ **应该这样做**:
- 搜索相关素材
- 改写成长文逻辑
- 自然融入章节
- 素材真实且相关
- 记录使用日志
