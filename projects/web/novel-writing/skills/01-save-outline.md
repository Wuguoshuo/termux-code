# Skill 1: 需求理解与大纲保存

> 收集小说设定，保存到 _briefs/ 文件夹
>
> 这是创建新小说项目的第一步

---

## 触发时机

1. 用户开始新小说项目时
2. 用户要求修改已有设定时

---

## 工作流程

### Step 1: 理解用户需求

**Think Aloud 示例**:
```
用户说："我想写一部关于时间旅行的科幻小说"

我思考：
- 这是新小说项目
- 需要收集完整的设定信息
- 需要了解：类型、题材、风格、目标读者等

我的计划：
1. 询问关键设定问题
2. 收集所有必要信息
3. 整理成设定文档
4. 保存到 _briefs/
```

**需要收集的信息**:

#### 基本信息
- 小说名称（或暂定名）
- 小说类型（玄幻/都市/科幻/悬疑/言情等）
- 题材（时间旅行/修仙/悬疑/爱情等）
- 风格基调（轻松/严肃/幽默/热血等）
- 目标读者（青少年/成年/全年龄）

#### 世界观设定
- 时间背景（古代/现代/未来）
- 地点（城市/星际/异界/校园等）
- 世界规则（修仙体系/科技规则/魔法设定等）
- 特殊设定（如有）

#### 主要人物设定
- 主角：
  - 姓名、年龄、外貌
  - 性格特点
  - 背景/来历
  - 核心目标/动机
  - 特殊能力（如有）

- 配角（3-5个重要配角）：
  - 姓名、与主角关系
  - 性格特点
  - 在故事中的作用

- 反派（如有）：
  - 姓名、动机
  - 与主角的冲突

#### 核心冲突
- 主要矛盾是什么？
- 外部冲突（谁和谁？）
- 内部冲突（主角的挣扎？）
- 预期结局方向

#### 写作规划
- 预计总字数
- 预计章节数
- 单章字数范围
- 更新频率（如有）

---

### Step 3: 确认并保存

**确认要点**:
- [ ] 小说类型明确？
- [ ] 世界观设定清晰？
- [ ] 主要人物设定完整？
- [ ] 核心冲突明确？
- [ ] 写作规划合理？

**保存操作**:
- 保存到 `_briefs/小说名-设定brief.md`
- 同时更新 `README.md`（添加小说列表）

## 特殊情况处理

### 情况 1: 用户信息不完整

**处理**:
- 使用 AskUserQuestion 工具询问缺失信息
- 可以根据类型提供默认建议
- 用户确认后创建文档

**示例**:
```
用户："写一个修仙小说"

我：
- 知道类型是修仙
- 但缺少：小说名、主角设定、世界观

需要询问：
1. 小说叫什么名字？（或暂时无题）
2. 主角设定？（姓名、年龄、性格）
3. 世界观设定？（修仙体系/等级等）
```

### 情况 2: 用户已有设定文档

**处理**:
- 读取现有设定文档
- 询问用户是否修改
- 如需修改，执行本 Skill 更新

### 情况 3: 用户要求修改设定

**处理**:
1. 读取现有设定文档
2. 理解用户修改要求
3. 更新相应部分
4. 保存新版本
5. 备份旧版本（如有必要）

创建设定文档前，自检：

- [ ] 小说类型清晰？
- [ ] 世界观设定完整？
- [ ] 主角设定有性格、背景、目标？
- [ ] 重要配角有设定？
- [ ] 反派有动机？
- [ ] 核心冲突明确？
- [ ] 写作规划合理（字数/章节）？

---

## 常见错误避免

❌ **不要这样做**:
- 设定过于抽象，缺乏具体细节
- 世界观逻辑自相矛盾
- 人物设定太单薄，只有外貌没有性格
- 主角没有明确的目标和动机
- 核心冲突不明确

✅ **应该这样做**:
- 世界观逻辑自洽
- 人物设定立体（外貌+性格+背景+目标）
- 主角有清晰的动机
- 核心冲突明确且有层次

**执行本 Skill 时，必须透明化思考**：

```

用户说："我想写一部关于时间旅行的科幻小说"

1. 询问缺失的关键信息
```
