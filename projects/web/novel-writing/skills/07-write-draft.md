# Skill 7: 创作章节初稿

> 基于讨论确认的方案，创作章节初稿
> 重点：先写出来，不追求完美，审校环节会优化

---

## 触发时机

1. 用户确认 Skill 3 的情节方案后
2. 用户要求修改已有章节时

---

## 工作流程

### Step 1: 确认方案

**Think Aloud 示例**:
```
用户选择了【方案 2：穿越者现身】

我确认：
- 核心情节：神秘人出现，制造悬念
- 主角反应：震惊、警惕、好奇
- 关键对话：揭示世界观和主角身世
- 收尾：留下悬念，为下章铺垫

我的计划：
1. 按照 Skill 3 确认的方案创作
2. 确保：
   - 对话自然
   - 场景具体
   - 人物行为符合人设
   - 节奏张弛有度

3. 写完后进入 Skill 8 审校
```

---

### Step 2: 写作原则

#### 核心原则

1. **故事驱动**
   - 每个情节都要推动故事发展
   - 不要写无关的流水账
   - 确保冲突和转折

2. **人物成长**
   - 主角要有变化（情绪/认知/能力）
   - 配角有发展
   - 人物关系动态变化

3. **具体场景**
   - 不用抽象描述
   - 用具体场景、对话、行动展现
   - "Show, don't tell"

4. **节奏张弛**
   - 快慢结合
   - 有紧张感也有放松
   - 读者不会疲劳

---

### Step 3: 章节结构

推荐结构：

#### 开场（10-15%）
- 场景引入（时间、地点、环境）
- 主角状态（在做什么、心情如何）
- 悬念引入（新情况/新问题）

#### 发展（50-60%）
- 冲突升级
- 人物互动
- 情节推进
- 细节描写

#### 高潮（20-25%）
- 最大冲突
- 关键转折
- 情感爆发

#### 收尾（5-10%）
- 本章结束
- 情绪平复
- 为下章铺垫（伏笔/悬念）

---

### Step 4: 写作要点

#### 1. 开头吸引人（3 秒法则）

✅ **好的开头**:
- 直接进入情节："林凡推开实验室的门..."
- 制造悬念："地上躺着一个人，穿着奇怪的银色服装..."
- 展示冲突："王德发冷笑一声，'你以为你能逃掉？'"

❌ **避免的开头**:
- 冗长描述："这天天气晴朗，万里无云，林凡走在去实验室的路上..."
- 空洞总结："林凡是一个有着坚定信念的年轻人..."
- 过度铺垫："在开始今天的故事之前，让我们回顾一下..."

#### 2. 情节有冲突

每个段落都要推动冲突：
- 外部冲突：人 vs 人、人 vs 环境
- 内部冲突：主角的挣扎、选择

**检查**:
- 这一章主角面临什么困难？
- 困难如何升级？
- 最终如何解决（或为后续埋下伏笔）？

#### 3. 人物行为符合人设

写作时时刻问自己：
- "这个角色会这样做吗？"
- "他的性格是X，他会说这样的话吗？"

**方法**:
- 保持角色一致性
- 避免OOC（Out Of Character）
- 通过行动和对话展现性格，而不是叙述

#### 4. 场景描写具体生动

✅ **具体描写**:
```
李明看着手机屏幕，手指悬在拨打键上。

三分钟前，他收到一条匿名短信："真相在地下室，去不去随你。"

他想起昨天的对话，那个戴黑帽子的神秘人说的话...

窗外的雨越下越大，敲打着玻璃，像有人在敲门。
```

❌ **抽象描述**:
```
李明陷入了沉思，心情复杂。

他想着昨天发生的事情，内心充满了矛盾和不安。

外面的天气很糟糕，更加重了他的焦虑感。
```

#### 5. 对话符合人物

**原则**:
- 每个人物有自己的说话方式
- 对话要推动情节
- 不要说教、不要信息倾倒

**示例**:
```
❌ 不好的对话：
神秘人："林凡，我告诉你一个秘密，时间机器是2010年发明的，
    原理是量子纠缠，你的父亲是发明者，现在反派要抢夺..."

✅ 好的对话：
神秘人："你父亲...他一直在等这一天。"
林凡："什么意思？"
神秘人："地下室那个机器，不是一般的设备。"
林凡："那是..."
神秘人："时间门。你的命门，也是。"
```

#### 6. 节奏张弛有度

**方法**:
- 紧张场景（战斗/追逐）→ 短句，快节奏
- 放松场景（对话/思考）→ 长句，慢节奏
- 交替使用，读者不会疲劳

**检查**:
- 连续紧张场景是否过长？（考虑插入轻松对话）
- 连续对话场景是否过长？（考虑插入动作）

## 特殊情况处理

### 情况 1: 用户要求修改已有章节

**流程**:
1. 读取已有章节（`drafts/` 或 `published/`）
2. 理解用户修改要求
3. 保留原有优点，修改不足部分
4. 不改变章节核心走向（除非用户明确要求）

**Think Aloud**:
```
用户要求修改第3章，希望加强对话...

我思考：
- 读取第3章原文
- 分析当前对话情况
  - 是否有大量说明性对话？
  - 是否缺少人物个性？
  - 是否对话不自然？

- 修改方向：
  - 减少信息倾倒
  - 增加人物互动
  - 用动作代替说明

计划：
1. 读取第3章
2. 加强对话部分
3. 保持原有情节不变
4. 保存修改后版本
```

### 情况 2: 用户指定字数

**处理**:
- 按用户要求调整字数
- 精简：删除冗余描写，加快节奏
- 扩充：增加细节描写，丰富对话

### 情况 3: 用户指定风格

**处理**:
- 调整写作风格（幽默/严肃/悬疑等）
- 但保持人物一致性
- 风格改变的是叙述方式，不是人设

## 质量检查清单

写作完成后，自检：

- [ ] 标题吸引人？
- [ ] 开头快速进入情节？
- [ ] 有明确的冲突？
- [ ] 人物行为符合人设？
- [ ] 场景描写具体？
- [ ] 对话自然，各有人物特色？
- [ ] 节奏张弛有度？
- [ ] 字数合理？（2000-5000字，根据小说类型）
- [ ] 结尾为下章铺垫？

---

## 常见错误避免

❌ **不要这样做**:
- 开头冗长，迟迟不进入情节
- 流水账，没有冲突和转折
- 人物OOC，行为不符合人设
- 抽象描写，缺乏具体场景
- 对话说教，大量信息倾倒
- 全章只有紧张或只有放松
- 超出用户确认的方案范围

✅ **应该这样做**:
- 开头快速吸引人
- 每段都推动情节
- 人物行为合理一致
- 用具体场景展现
- 对话自然且有人物特色
- 快慢节奏结合
- 严格按照确认的方案写作

## Think Aloud 要求

**执行本 Skill 时，必须透明化思考**：

```
示例：

用户确认选择方案 2，要求 3000 字...

我思考：
- 确认方案 2 的核心情节
  - 场景：实验室

  - 对话：揭示世界观
  - 收尾：悬念

- 写作要点：

  - 开场：15%（450字）
  - 发展：55%（1650字）
  - 高潮：25%（750字）

计划：
1. 按照 55:25:5 结构写作
2. 确保对话质量和悬念设置
3. 完成后保存到 drafts/
```
