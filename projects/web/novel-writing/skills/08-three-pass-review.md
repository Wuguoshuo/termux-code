# Skill 8: 三遍审校（降 AI 味）

> **最关键的一步**：系统化降低 AI 检测率，增加"人味"
>
> 三遍审校：内容审校 → 风格审校 → 细节打磨

---

## 触发时机

1. 完成章节初稿后（Skill 7）
2. 用户要求审校已有章节时

---

## 工作流程

### Step 1: 读取章节

**Think Aloud 示例**:
```
用户要求审校第3章...

我思考：
- 需要读取第3章内容
- 了解章节的情节和风格
- 执行三遍审校

我的计划：
1. 读取 drafts/第3章.md
2. 执行第一遍：内容审校
3. 执行第二遍：风格审校（降 AI 味核心）
4. 执行第三遍：细节打磨
5. 生成审校报告
```

---

## 第一遍：内容审校

### 目标
检查逻辑、事实、结构的准确性

### 检查清单

#### 1. 逻辑清晰
- [ ] 前后无矛盾？
- [ ] 因果关系合理？
- [ ] 没有逻辑跳跃？

**常见问题**:
```
❌ 矛盾示例：
第2段说："李明今天没有带手机"
第10段说："李明拿起手机拨打电话"
→ 逻辑矛盾

✅ 修改：
第2段："李明的手机在桌上，但他不敢拿"
或第10段："李明借了同事的手机"
```

#### 2. 人物行为合理
- [ ] 符合人设？（性格、能力、状态）
- [ ] 行为有动机？
- [ ] 没有OOC（Out Of Character）？

**检查方法**:
- 时刻问："这个角色会这样做吗？"
- 对比前文，保持一致性

#### 3. 情节连贯
- [ ] 没有莫名其妙的事件？
- [ ] 每个情节都有铺垫或解释？
- [ ] 没有突兀的跳跃？

#### 4. 无硬伤
- [ ] 时间线正确？
- [ ] 地点逻辑合理？
- [ ] 常识性错误？
- [ ] 世界观一致？（不违反设定）

**常见硬伤**:
```
❌ 时间问题：
"现在是早上8点，但天已经黑了" → 矛盾

❌ 地点问题：
"他在北京，突然出现在上海" → 缺少解释

❌ 常识问题：
"他跑了几十公里，只用了5分钟" → 不合理

❌ 世界观问题：
设定说"普通人不会法术"，但路人突然释放法术 → 违反设定
```

#### 5. 结构合理
- [ ] 不跑题？
- [ ] 有明确的开头、发展、高潮、收尾？
- [ ] 段落顺序合理？

## 第二遍：风格审校（降 AI 味核心）

### 目标
删除 AI 腔，增加人味

### 检查清单

#### 1. 删除套话

**必须删除的套话**:
```
❌ "在当今时代"
✅ 直接写场景/时间

❌ "综上所述"
✅ "所以..."或直接开始下一段

❌ "值得注意的是"
✅ 直接说重要内容

❌ "经过充分的思考"
✅ "想了很久"或直接描述思考内容

❌ "毋庸置疑"
✅ "确实"或用具体证据支撑

❌ "从...角度来看"
✅ 直接说观点

❌ "从某种意义上来说"
✅ 直接表达
```

#### 2. 拆解 AI 句式

**避免连续使用以下句式**:
```
❌ "不是...而是..."（连续出现）
✅ 改写为其他句式：
   - "更像是..."
   - "其实是..."
   - "..."

❌ "不仅...而且..."
✅ 改写：
   - 拆分成两句
   - "不仅"去掉
   - 用动作展现而非陈述

❌ "一方面...另一方面..."
✅ 直接说两面，不用这个框架

❌ "从...到...，无一例外"
✅ 删掉，直接描述

❌ "无论是...还是..."（滥用）
✅ 根据场景选择
```

#### 3. 替换书面词汇

**书面词汇 → 口语/具体化**:
```
❌ "显著提升"
✅ "提升了一倍" / "快了三倍" / 具体效果

❌ "充分利用"
✅ "用好" / "用起来"

❌ "进行操作"
✅ 直接用动词："操作" / "设置" / "调整"

❌ "产生深远的影响"
✅ "改变了..." / "让...不一样"

❌ "采取行动"
✅ "动手" / "开始行动" / "往前走"

❌ "进行分析"
✅ "分析" / "想想"

❌ "做出决定"
✅ "决定" / "选了"

❌ "进行思考"
✅ "想" / "琢磨"

❌ "实现...的目标"
✅ "达成了目标" / "做到了"

❌ "具有...的特征"
✅ "的特点是..." / "...样"

❌ "在...的过程中"
✅ "...时" / 删掉

❌ "随着...的发展"
✅ "..."直接说变化

❌ "呈现出...的态势"
✅ "..."直接描述
```

#### 4. 改成口语化

**原则**: 口语不是粗俗，是自然、简洁

```
❌ 正式：
"他进行了详细的思考后，决定采取相应的行动"
✅ 口语：
"想了想，他决定..."

❌ 正式：
"她感到一种难以言喻的悲伤涌上心头"
✅ 口语：
"她心里堵得慌，像有什么东西碎了"

❌ 正式：
"这个决定具有极其重要的意义"
✅ 口语：
"这个决定改变了所有"

❌ 正式：
"他们进行了深入的探讨"
✅ 口语：
"他们聊了很久"

❌ 正式：
"通过这个事件，我们可以得出一个结论"
✅ 口语：
"这件事证明了..."
```

#### 5. 加入真实细节

**抽象表达 → 具体场景/对话**:
```
❌ 抽象：
"他很生气"
✅ 具体：
"他拳头握得发白，额头上青筋暴起"

❌ 抽象：
"天气很恶劣"
✅ 具体：
"雨下得像要把城市淹了，风把路边的树都吹弯了"

❌ 抽象：
"房间很乱"
✅ 具体：
"衣服扔在沙发上，外卖盒堆在桌上，书撒了一地"

❌ 抽象：
"他们聊得很开心"
✅ 具体：
"李明笑得前仰后合，王强拍着桌子喊'再来一个！'"
```

#### 6. 加入个人态度

**中立客观 → 明确观点/情感**:
```
❌ 中立 AI 腔：
"这个问题的解决需要多方面的考虑，不同的人可能有不同的看法"

✅ 个人态度：
"我觉得这样做不对..."
"他肯定不答应"
"这事儿没法谈"

❌ 中立：
"经过讨论，双方达成了共识"

✅ 个人：
"终于说通了，大家都松了一口气"

❌ 中立：
"从长远来看，这可能产生积极的影响"

✅ 个人：
"这肯定会有用"
"长远看，这样做是对的"
```

### 第二遍具体操作

#### 方法 1: 逐段审校

**对每一段**:
1. 识别 AI 腔
2. 判断问题类型（套话/句式/书面词）
3. 改写
4. 重新阅读，确认

#### 方法 2: 大声朗读

**朗读时注意**:
- 是否顺口？
- 哪里别扭？
- 哪里像机器说的？
- 哪里像真人说的？

#### 方法 3: 对比参考

**如果用户提供了风格指南或历史文章**:
- 对比风格
- 模仿语气
- 模仿句式

## 第三遍：细节打磨

### 目标
优化标点、排版、节奏

### 检查清单

#### 1. 句子长度合适

**原则**:
- 主要句子：15-25 字
- 超长句子：不超过 30 字

**操作**:
- 找出超过 30 字的长句
- 拆分成 2-3 个短句

**示例**:
```
❌ 超长句：
"林凡站在窗前，看着窗外的雨下得越来越大，心里想着爷爷临终前说的话，眼泪不知不觉地流了下来"

✅ 拆短：
"林凡站在窗前。外面的雨越下越大。

爷爷临终前的话，一遍遍在脑子里转。

眼泪不知不觉流了下来。"
```

#### 2. 段落不太长

**原则**:
- 手机屏幕：3-5 行
- 电脑屏幕：5-8 行
- 不要一大段到底

**操作**:
- 超长段落拆分
- 按情节、场景、对话分段

#### 3. 标点自然

**原则**:
- 多用句号，少用逗号连接长句
- 引号、省略号正确使用
- 标点符合中文规范

**检查**:
- [ ] 没有连续逗号？
- [ ] 句号使用充分？
- [ ] 对话标点正确？

#### 4. 节奏有变化

**原则**:
- 快慢结合
- 张弛有度
- 读者不会疲劳

**检查**:
- [ ] 紧张场景后有没有放松？
- [ ] 连续对话后有没有动作描写？
- [ ] 句式有变化？（短/长句混合）

---

### 第三遍具体操作

#### 方法 1: 大声朗读

**朗读时注意**:
- 哪里读得快？（紧张）
- 哪里读得慢？（放松）
- 哪里停顿别扭？（标点问题）
- 哪里一口气读不完？（句子太长）

#### 方法 2: 视觉检查

**屏幕显示**:
- 段落长度是否合适？
- 有没有太密集的段落？
- 有没有太多的空行？

## 特殊情况处理

### 情况 1: 用户只要求某一遍

**处理**:
- 只执行指定的审校遍数
- 不执行其他遍数

**示例**:
```
用户：只做风格审校

我：
- 跳过第一遍
- 执行第二遍：风格审校
- 跳过第三遍
- 输出第二遍结果
```

### 情况 2: 用户要求重点检查某方面

**处理**:
- 三遍都执行
- 但在相应遍数重点关注用户指定的方面

**示例**:
```
用户：重点检查人物行为和对话

我：
- 第一遍：重点关注人物行为
- 第二遍：重点关注对话风格
- 第三遍：重点关注对话标点和节奏
```

## 常见错误避免

❌ **不要这样做**:
- 只做一遍审校
- 只改标点不改内容
- 删除套话但不改写句式
- 改写但失去原意
- 不 Think Aloud

✅ **应该这样做**:
- 严格执行三遍审校
- 每一遍有明确目标
- 降 AI 味要彻底
- Think Aloud 说明每一步
- 保留原意的同时改写

---

## Think Aloud 要求

**执行本 Skill 时，必须透明化思考**：

```
示例：

用户要求审校第3章...

我思考：
- 需要执行三遍审校
  - 第一遍：内容审校（逻辑、人物、事实）
  - 第二遍：风格审校（降 AI 味核心）
  - 第三遍：细节打磨（标点、排版、节奏）

计划：
1. 读取第3章
2. 第一遍：检查逻辑、人物、事实
3. 第二遍：删除套话、改写 AI 句式、口语化
4. 第三遍：调整句子长度、段落长度、标点、节奏
5. 生成完整审校报告
6. 保存修改后版本
```
