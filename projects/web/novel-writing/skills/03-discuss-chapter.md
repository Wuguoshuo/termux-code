# Skill 3: 选题与章节讨论

> **核心原则**: 不直接写，先讨论！避免方向错误造成大量返工

---

## 触发时机

用户要求写新章节时触发。

---

## 工作流程

### Step 1: 读取上下文

我需要先了解：
1. **小说设定** - 读取 `_briefs/小说名-设定brief.md`
2. **前文内容** - 读取前几章，了解情节进度
3. **人物关系** - 了解当前人物状态

**Think Aloud 示例**:
```
用户要求写第3章...

我需要先了解上下文：
- 读取设定文档，了解世界观和主角
- 读取第1、2章，了解当前情节进度
- 确认主角当前的状态和目标

我的计划：
1. 读取设定文档
2. 读取前文内容
3. 执行 Step 2，提供情节方案
```

---

### Step 2: 提供 3-4 个情节方案

**核心原则**：
- 提供多个选项，让用户选择
- 每个方案都要完整（标题、情节、冲突、人物发展）
- 包含工作量评估和优劣势分析
- **等待用户确认**，不要假设或自作主张

**每个方案必须包含**:

#### 1. 章节标题（吸引人）
- 简洁有力
- 暗示核心冲突
- 符合整体风格

#### 2. 核心情节（3-5 个主要事件）
按时间顺序排列：
- 事件 1：开场/引入
- 事件 2：冲突升级
- 事件 3：转折点
- 事件 4：高潮
- 事件 5：收尾/铺垫下一章

#### 3. 关键冲突和转折
- 主要冲突是什么？（外部冲突 / 内部冲突）
- 转折点在哪里？（意料之外但情理之中）
- 矛盾如何升级？

#### 4. 人物发展
- 主角在本章有什么变化？
- 配角有什么成长？
- 人物关系如何变化？
- 伏笔如何埋下？

#### 5. 工作量评估
- 评级（☆）：
  - ★☆☆☆☆：简单，以对话为主
  - ★★☆☆☆：中等，有动作场面
  - ★★★☆☆：复杂，多条线索并行
  - ★★★★☆：困难，需要详细描写
  - ★★★★★：极难，涉及世界观设定或复杂战斗

- 预计字数：2000-5000 字（根据小说类型）

#### 6. 优势和劣势
- ✅ 优势：这个方案的好处（情节紧凑、人物突出、节奏好等）
- ❌ 劣势：可能的不足（需要更多背景、节奏慢、人物少等）

---

### Step 3: 等待用户选择

**核心原则**：
- 提供 3-4 个方案后，**停止**
- 等待用户明确选择哪个方案
- 不要假设用户会选哪个
- 如果用户要求修改，按用户要求调整方案

**示例对话**:
```
我：为您准备了 3 个方案：

【方案 1】时间裂隙
核心情节：主角在地下室发现时间机器...

【方案 2】穿越者现身
核心情节：神秘人出现，声称来自未来...

【方案 3】真相初现
核心情节：主角发现时间机器的制造者是自己...

请告诉我您选择哪个方案？或者您有其他想法？
```

---

### Step 4: 确认方案并进入 Skill 7

用户选择方案后：
1. 确认用户选择
2. 说明下一步计划
3. 进入 Skill 7 创作初稿

**示例对话**:
```
我：好的，您选择了【方案 2：穿越者现身】

这个方案的核心是：
- 神秘人出场，制造悬念
- 通过对话揭示世界观
- 引出主要反派

我的下一步计划：
- 执行 Skill 7，根据方案 2 创作第 3 章初稿
- 确保悬念和对话质量

可以开始吗？
```

## 特殊情况处理

### 情况 1: 用户给出明确要求

如果用户已经明确指出情节方向：
- 不需要提供多个方案
- 直接进入 Skill 7，按用户要求写作
- 但仍需 Think Aloud 说明理解

**示例**:
```
用户：写第3章，主角在实验室发现时间机器，按下按钮后穿越到2024年

我：
思考：
- 用户已经明确给出了情节框架
  - 地点：实验室
  - 事件：发现时间机器
  - 结果：穿越到2024年
- 不需要提供多个方案
- 直接进入 Skill 7 创作初稿

计划：
1. 确认理解
2. 执行 Skill 7，创作第 3 章
3. 确保穿越场景描写生动
```

### 情况 2: 用户要求讨论而非选择

如果用户说"帮我讨论一下"：
- 深入分析每个方案的优缺点
- 提供建议，但不做决定
- 让用户自己权衡

### 情况 3: 用户要求修改方案

如果用户说"方案2不错，但改成..."：
- 修改方案 2，按用户要求调整
- 重新输出修改后的方案
- 确认后再进入 Skill 7

## 质量检查清单

提供方案前，自检：

- [ ] 标题吸引人？
- [ ] 情节连贯？（符合前文逻辑）
- [ ] 冲突明确？（不是流水账）
- [ ] 人物有发展？（不是原地踏步）
- [ ] 有伏笔？（为后续铺垫）
- [ ] 工作量评估合理？
- [ ] 优劣势分析客观？

---

## 常见错误避免

❌ **不要这样做**:
- 直接开始写作，不提供方案
- 假设用户会选哪个，直接写
- 提供方案后不等用户确认就继续
- 方案太空洞，没有具体情节

✅ **应该这样做**:
- 先提供 3-4 个完整方案
- 等待用户明确选择
- Think Aloud 说明思考过程
- 每个方案都要具体、可执行

---

## Think Aloud 要求

**执行本 Skill 时，必须透明化思考**：

```
示例：

用户要求写第3章...

我思考：
- 首先需要读取上下文
  - 设定文档：了解世界观和主角
  - 前2章：了解情节进度
  - 当前人物状态

- 然后设计 3 个情节方案
  - 方案1：主角发现时间机器（符合用户预期）
  - 方案2：神秘人出现（制造悬念）
  - 方案3：回忆杀（揭示背景）

- 每个方案都要包含：
  - 标题、情节、冲突、人物发展
  - 工作量评估、优劣势

- 最后等待用户选择

我的计划：
1. 读取设定文档
2. 读取前2章
3. 提供 3 个方案
4. 等待用户确认
```
